= Projects

The Mifos ecosystem is composed of several interconnected repositories, each serving a distinct role in the financial inclusion platform. At the center is Apache Fineract, the core banking engine that provides the REST API for loan management, savings, client management, and accounting. Surrounding it are community applications, mobile SDKs, and operational tooling.

The typical Mifos project is a multi-module Gradle build. The root project defines shared configuration through convention plugins, while individual modules encapsulate bounded contexts or deployable artifacts. Below is a representative project layout:

The relationship between these projects is important to understand. The `fineract-provider` module depends on `fineract-infrastructure`, `fineract-accounting`, and `fineract-portfolio`. The `community-app` consumes the REST API exposed by `fineract-provider`. Mobile applications interact exclusively through the public API surface, never directly with the database.

Each module has its own `build.gradle.kts` that applies one or more convention plugins from `buildSrc`. This ensures, for example, that every Java module inherits the same Checkstyle configuration, JaCoCo coverage thresholds, and dependency version constraints without any copy-paste.

Key layout rules:

* **Production code** lives under `src/main/java/` with the package root `org.mifos`.
* **Test code** mirrors the production package structure under `src/test/java/`.
* **Integration tests** may be placed in `src/integrationTest/java/` with a corresponding source set.
* **Resources** including Spring configuration, i18n bundles, and Flyway migrations go under `src/main/resources/`.
* **Test resources** override or supplement production configuration under `src/test/resources/`.

WARNING: Never place test classes in the production source tree. The Gradle build will package them into the deployed JAR, inflating artifact size and potentially leaking test utilities into production.
