= Continuous Integration

The Mifos CI philosophy is built on a single principle: **every pull request must pass CI before it can be merged.** This ensures that the `main` branch is always in a known-good state, deployable at any time.

Key CI tenets:

* **Fast feedback**: The CI pipeline should complete in under 15 minutes. Developers should not have to wait hours to know if their PR is ready for review.
* **Fail fast**: Run the fastest checks first (formatting, Checkstyle) and the slowest checks last (integration tests, security scans). This way, trivial issues are caught quickly.
* **Deterministic**: The same commit should produce the same CI result every time. Flaky tests are treated as P0 bugs and are either fixed immediately or quarantined.
* **Composable**: CI steps are organized as independent jobs that can run in parallel. The overall pipeline is a directed acyclic graph (DAG), not a linear sequence.

The typical CI pipeline stages:

. **Format** — Spotless check (< 1 min)
. **Static Analysis** — Checkstyle, PMD, SpotBugs, Error Prone (< 5 min)
. **Unit Tests** — `./gradlew test` (< 10 min)
. **Integration Tests** — `./gradlew integrationTest` (< 15 min)
. **Security Scan** — Snyk, Grype, Semgrep (< 5 min)
. **Coverage Report** — JaCoCo report upload (< 2 min)

== Github Actions

GitHub Actions is the CI/CD platform for Mifos repositories. Every repository includes a `.github/workflows/ci.yml` file that defines the pipeline. Below is a representative Mifos CI workflow:

[source,yaml]
----
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

permissions:
  contents: read
  security-events: write

env:
  GRADLE_OPTS: >-
    -Dorg.gradle.daemon=false
    -Dorg.gradle.parallel=true
    -Dorg.gradle.caching=true
    -Dorg.gradle.jvmargs=-Xmx2g

jobs:
  format:
    name: Format Check
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '25'
      - name: Check formatting
        run: ./gradlew spotlessCheck

  static-analysis:
    name: Static Analysis
    runs-on: ubuntu-latest
    needs: format
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '25'
      - name: Run Checkstyle, PMD, SpotBugs
        run: ./gradlew checkstyleMain checkstyleTest pmdMain pmdTest spotbugsMain spotbugsTest

  test:
    name: Unit Tests
    runs-on: ubuntu-latest
    needs: format
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '25'
      - name: Run tests with coverage
        run: ./gradlew test jacocoTestReport jacocoTestCoverageVerification
      - name: Upload coverage report
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: '**/build/reports/jacoco/'

  integration-test:
    name: Integration Tests
    runs-on: ubuntu-latest
    needs: test
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '25'
      - name: Run integration tests
        run: ./gradlew integrationTest

  security:
    name: Security Scan
    runs-on: ubuntu-latest
    needs: format
    steps:
      - uses: actions/checkout@v4
      - name: Run Snyk security scan
        uses: snyk/actions/gradle@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
        with:
          args: --severity-threshold=high
      - name: Run Semgrep
        uses: semgrep/semgrep-action@v1
        with:
          config: >-
            p/java
            p/owasp-top-ten
----
